nmap -sV -sC -oN nmap 3.127.234.70

PORT   STATE SERVICE VERSION
22/tcp open  ssh     OpenSSH 7.6p1 Ubuntu

Two open ports were found:

  • 22/tcp - SSH
  • 80/tcp - HTTP

Web Server

The default Apache page was displayed. Nothing interesting there. I started directory brute-forcing:

gobuster dir -u http://3.127.234.70/ -w /usr/share/dirb/wordlists/common.txt

Found the /secret directory, but still nothing useful. Ran another scan:

gobuster dir -u http://3.127.234.70/secret/ -w /usr/share/dirb/wordlists/common.txt

Found these:

  • /company
  • /admin.html
  • /root
  • /test
  • /tools

Privilege Escalation

I found a file named TrollFace.jpg in /var/www.

Using steghide, I extracted a hidden password.

I then switched to user alex using that password.

Next, I discovered that vim can be run with sudo privileges:

sudo vim -c ':! /bin/sh'

And that gave root access.

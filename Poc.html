Proof of Concept: CORS Misconfiguration

This code attempts to fetch data from:

https://api.medilink.healthatom.com/api/v1/citas

with credentials (cookies, authorization headers) automatically included if you are logged into Healthatom in the same browser.

Note: In modern browsers, if the server responds with Access-Control-Allow-Origin: * together with Access-Control-Allow-Credentials: true, the browser will block reading the response and an error will appear in the console. If the data is successfully displayed, it means the server is not enforcing the restrictions correctly and the vulnerability is exploitable.

Result will appear here...

Interpreting the results:

  • If data appears: The vulnerability is fully exploitable – any malicious website can steal your information.
  • If an error appears in the console (e.g., "CORS ..."): The browser prevented reading the data due to the unsafe configuration, but the misconfiguration still exists and might be a risk in other contexts (e.g., native apps).
  • If another error appears (e.g., 401 Unauthorized): The session might be inactive or authentication is required.
